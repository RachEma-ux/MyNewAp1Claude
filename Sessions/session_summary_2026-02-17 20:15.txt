- UI: category filter dropdown, category/subcategory column in table, capability chip selector in create/edit dialog

### Extended to 5 Entry Types (Provider, LLM, Model, Agent, Bot)
Previously only "provider" and "model". Now supports all 5 types with full definitions:

1. **Provider** — Infrastructure access layer (Cloud API, Local Runtime, Custom Adapter, Edge, Air-Gapped, Mock, Archived)
2. **LLM** — Technology class (General Purpose, Code, Multimodal, Domain Specific, Small Language Model)
3. **Model** — Trained instance (Base LLM, Embedding, Speech, Fine-Tuned, Adapter, Quantized, Distilled, Experimental, Composite)
4. **Agent** — Orchestrated intelligent entity (Conversational, Task Executor, Orchestrator, Autonomous, Specialist)
5. **Bot** — Lightweight automated responder (Chat Bot, Command Bot, Notification Bot, Integration Bot, Q&A Bot)

Each type has:
- Detailed JSDoc comment with definition, characteristics, and examples
- Category map with subcategories
- Appropriate capabilities (functional, operational, governance, meta)
- Dedicated icon in UI (Server, Brain, Package, Workflow, Bot)

### Capabilities Extended
New capabilities for new types: memory, multi_step, coordination, channel_binding, long_running, lightweight, policy_bound

### Types Definition Document
Created `Types Definition.md` with full definitions, comparison table, and architectural stack diagram.
